API key

Use this first to get a reusable playground API key and Ezofis token for the rest of the APIs.

Endpoints: POST /api/Client/apiKey and GET /api/Client/apiKey

Query params: userName, password

Auth: no X-API-Key header for this route.

POST /api/Client/apiKey?userName=admin@ezofis.com&password=your_password
{
  "tenantId": "123",
  "token": "eyJhbGciOi...",
  "apiKey": "x4iDIG6..."
}

QR code

Generates a QR code image from text and returns Base64 content in output.

Endpoint: POST /api/qrcode/generate

Headers: Content-Type: application/json, X-API-Key: <apiKey>

{
  "qrvalue": "INV-100245"
}
{
  "output": "iVBORw0KGgoAAAANSUhEUgAA...",
  "id": 1,
  "EncryptOutput": null
}

Form details

Fetches forms from Ezofis using query filters, pagination, and optional encryption modes.

Endpoint: POST /api/FormDetails

Headers: X-API-Key, Ezofis-Token

{
  "query": {
    "filterBy": "",
    "sortBy": "formId desc",
    "pageNo": 1,
    "pageSize": 10
  },
  "useEncryptedRequest": true,
  "useEncryptedResponse": true
}

When successful, the API returns id: 1 and either plain JSON in output or encrypted content depending on flags.

SubForm details

Returns available subform/panel titles for a form. Use these titles in the SubForm Fields API.

Endpoint: POST /api/SubFormDetails

Headers: X-API-Key, Ezofis-Token

{
  "formId": 139
}

Use the returned subform title strings exactly as-is when calling /api/SubFormFields.

SubForm fields

Gets field metadata for one or more selected subform titles.

Endpoint: POST /api/SubFormFields

Headers: X-API-Key, Ezofis-Token

{
  "formId": 139,
  "subforms": [
    "009 - Account Details"
  ]
}

If no fields are found or the title does not match, you may receive an empty result in output.

Generate PDF

Submits subform values and triggers archive/PDF generation in one API call.

Endpoint: POST /api/SubFormSubmitAndArchive

Headers: X-API-Key, Ezofis-Token

{
  "formId": 139,
  "templateId": 132,
  "entryIdForFormEntry": 0,
  "entryId": 0,
  "isMultiple": false,
  "panelName": "009 - Account Details",
  "fields": {
    "ed3dZq8mL_6QgGV26PAJj": "123456"
  }
}

Required fields: formId, templateId, panelName, and object fields.

Salesforce data

Sends a payload to Salesforce integration flow and returns normalized/decrypted output.

Endpoint: POST /api/GetDataFromSalesforce

Headers: X-API-Key, Ezofis-Token

{
  "payload": {
    "searchText": "ACC001"
  }
}
{
  "output": "{ ... salesforce response ... }",
  "id": 1,
  "EncryptOutput": null
}

Validation error example: payload is required.
